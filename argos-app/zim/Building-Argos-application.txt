====== Building ARGOS from the Ashes ======
Created Thursday 09 October 2025

===== Introduction: The Journey from IOTRAC to ARGOS =====

==== The Beginning: IOTRAC University Project ====

In early 2025, **IOTRAC** (IoT, Track, Recording, Act and Control) was conceived as a university scientific initiation project. The vision was clear: create a mobile application that would democratize IoT security, putting cybersecurity protection directly in the hands of users rather than relying solely on large corporations to secure their connected devices.

The core concept was revolutionary yet practical: implement a **cryptographic security layer** for IoT device communications, intercepting and protecting data traffic in real-time. To demonstrate this capability at the **NEXT Technology Event**, the team planned to build a 3D-printed mini autonomous car controlled by a Raspberry Pi, simulating a live hacking attack and demonstrating how the application could block it in real-time.

==== The Team and the Challenge ====

A collaborative team of **three developers** worked together, united by the belief in the project's innovative potential. The goal was ambitious: showcase a future where IoT devices—increasingly integrated into daily life—could be secured by an accessible, user-friendly mobile application.

Despite reaching the **final elimination round** of the NEXT, the team did not achieve classification. Multiple factors contributed:
* **Incomplete product development**: The solution required more refinement to be market-ready
* **Vision misalignment**: Team members had different perspectives on work methodology and application direction
* **Collaboration challenges**: Differences in commitment levels and technical approaches created friction

Following this outcome, the team mutually decided to part ways, with each member pursuing their own path.

==== The Rebirth: From Failure to Evolution ====

**Kiron García**, the project's founder and lead developer, remained deeply passionate about the original vision. Rather than abandoning the concept after the competition setback, a critical period of reflection and reevaluation began:

**What worked well:**
* ✅ Core cryptographic architecture (AES-256 + HMAC)
* ✅ Multi-protocol support (UDP/TCP/Bluetooth)
* ✅ FastAPI backend infrastructure
* ✅ React Native mobile foundation
* ✅ Authentication system implementation

**What needed transformation:**
* 🔄 Project branding and market positioning
* 🔄 Feature set and user experience
* 🔄 Technology stack modernization
* 🔄 Architecture reorganization
* 🔄 Product focus and target audience

==== The ARGOS Vision: A Complete Rework ====

The decision was made: **IOTRAC would evolve into ARGOS**—a complete reworking that preserves the solid technical foundation while introducing:

* **New Identity**: Fresh branding, modern UI/UX, professional positioning
* **Enhanced Features**: AI-powered threat detection, advanced monitoring, multi-device management
* **Expanded Technology**: Integration with modern AI/LLM systems, improved Bluetooth support
* **Refined Architecture**: Clean separation of concerns, scalable design patterns
* **Clear Direction**: Focused mission with well-defined user personas and use cases

This documentation marks the **starting point of the ARGOS transformation**—a comprehensive record of what exists today and the roadmap for what will be built tomorrow.

**Authentication Flow Implementation:**
{{{code: lang="typescript" linenumbers="True"
// Complete Authentication Journey
1. User enters email/password → /login
2. Backend validates credentials
3. If new user → /verify-email (email code)
4. If 2FA enabled → /verify-2fa (SMS/email code)
5. If TOTP setup required → /totp-setup (QR scan)
6. Success → /home (authenticated area)

// Token Management
- Access Token: Stored in AsyncStorage, sent in Authorization header
- Refresh Token: Stored securely, used for token renewal
- Auto-refresh: Interceptor automatically refreshes expired tokens
- Logout: Clears all tokens and redirects to login
}}}
