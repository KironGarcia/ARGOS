Created Sunday 05 January 2025

===== Current Situation Summary =====

Motor control system completely non-responsive despite all logical components appearing functional. Time-critical resolution needed today.

==== Confirmed Working Components ====
* ✅ **Raspberry Pi GPIO**: Blinky test successful with same jumper cables
* ✅ **Jumper Cable Integrity**: LED control works perfectly through same cables
* ✅ **Motors Physical Condition**: Tested individually before connection - both rotate
* ✅ **L298N Power Reception**: LED indicator ON when powered
* ✅ **Power Supply**: 5V stable from powerbank, GND connected via cut cable
* ✅ **Code Logic**: Pin assignments verified, control signals correct

==== EUREKA! ====
**Root Cause Discovered: L298N Power Configuration Issue**

The mystery has been solved! The problem was not in the components themselves, but in the **L298N module power configuration**. The 5V_EN jumper was causing a power supply conflict that prevented the logic circuitry from receiving adequate voltage.

===== The L298N 5V_EN Jumper Problem =====

**What Was Happening:**
* L298N module had 5V_EN jumper **INSTALLED** (factory default)
* With jumper installed, +5V pin becomes an **OUTPUT** (not input)
* Internal 78M05 regulator tries to generate 5V from +12V input
* **Critical Issue**: 5V input to +12V pin is insufficient for regulator (needs minimum 6-7V)
* Result: Logic circuitry receives inadequate power despite LED being ON

**Why LED Was ON But Nothing Worked:**
* Power LED connects directly to +12V input (before regulator)
* LED lights up with 5V input, creating false indication of proper power
* Internal logic circuits (controlling IN1-IN4) starved of proper 5V supply
* GPIO signals from Raspberry Pi were being ignored by unpowered logic

===== The Solution: Dual Power Supply Configuration =====

**Step 1: Remove 5V_EN Jumper**
* Physically remove the small jumper cap from 5V_EN pins
* This changes +5V pin from OUTPUT to INPUT mode

**Step 2: Dual Power Connection**

{{{code: lang="text" linenumbers="True"
Powerbank 5V (+) → L298N "+5V" pin    (Logic Power Supply)
Powerbank 5V (+) → L298N "+12V" pin   (Motor Power Supply) 
Powerbank GND (-) → L298N "GND" pin    (Common Ground)
}}}

**Why This Works:**
* +5V pin directly powers internal logic circuits (IN1-IN4 processing)
* +12V pin powers H-bridge motor drivers (OUT1-OUT4 outputs)  
* Both circuits receive clean, stable 5V power
* No voltage regulator conflicts or insufficient input voltage

===== Verification Results =====

**Power Measurements:**
* +5V pin: 5.0V (logic circuits powered)
* +12V pin: 5.0V (motor circuits powered)
* OUT1-OUT4: ~3.0V output (5V input minus 2V internal drop)

**Functional Test:**
* GPIO signals IN1-IN4 now properly recognized
* Motor outputs OUT1-OUT4 providing adequate drive voltage
* Motors responding correctly to direction and speed commands

===== Common Ground Connection Issue =====

**Problem:** Raspberry Pi and L298N need common ground reference, but single AWG 26 jumper cable insufficient for total current (~1A).

**Solution:** Multiple GND pins connection for current distribution and safety.

{{{code: lang="text" linenumbers="True"
Cable USB cortado (GND) → 3 jumpers paralelos:
                          ├── Jumper 1 → Pi Pin 6 (GND) — L298N common ground
                          ├── Jumper 2 → Pi Pin 9 (GND) — HC-SR04 ultrasonic (sensor cable)
                          └── Jumper 3 → Pi Pin 14 (GND) — L298N common ground
}}}

**Why This Works:** All Raspberry Pi GND pins internally connected to same ground plane. Using multiple pins distributes current load safely.
