===== Tests Performed =====

==== Test 1: Speed Calibration (Asymmetric) ====

**Purpose:** Identify and compensate Motor B delay issue

**Initial Configuration:**
* Motor A: 5% → 30%
* Motor B: 5% → 30%
* Result: ❌ Motor B had ~3 second delay compared to Motor A

**Problem Identified:**
* **Dead Zone Issue:** At low PWM (<15%), Motor B does not have sufficient torque to overcome internal friction
* Motor B consistently slower to respond than Motor A at identical PWM values
* Suspected causes:
  - Motor internal resistance differences
  - Dead zone threshold variation between motors
  - Physical friction asymmetry

**Solution Applied - Asymmetric Calibration:**
```
Motor A: 20% → 40% PWM
Motor B: 30% → 50% PWM (10% compensation offset)
```

**Result:** ✅ Improved synchronization - Motors start more simultaneously

**⚠️ Issues Identified:**

===== Problem 1: Excessive Rear Wheel Slippage =====
* **Observation:** Rear wheels slide/skid significantly during turns
* **Root Cause:** 3D printed wheels have insufficient traction on smooth surfaces
* **Impact:** 
  - Reduced turning efficiency
  - Unpredictable turning radius
  - Difficulty starting movement from standstill

**Solution Required:**
* Add rubber/foam material to rear wheels for increased friction

* Front wheels should remain low-friction (allow sliding during turns)

===== Problem 2: Difficulty Starting from Standstill =====
* **Observation:** Car struggles to begin moving, wheels spin without traction
* **Root Cause:** Low friction between 3D printed wheels and floor
* **Solution:** Same as Problem 1 - add traction material to rear wheels

===== Motor Asymmetry Analysis =====

==== Confirmed Issue: Motor B Delayed Response ====

**Symptoms:**
* Motor B takes longer to initiate rotation (~3 seconds at low PWM)
* Motor B appears slower than Motor A at identical PWM values
* Issue more pronounced at PWM <30%

**Investigation Results:**

**Hardware Checks Performed:**
* ✅ L298N connections verified (OUT3, OUT4 correct)
* ✅ Power cables checked (no loose connections)
* ✅ Jumpers verified (ENA, ENB properly installed)
* ✅ Motor physical condition checked (both motors new)

**Possible Causes:**
1. **Dead Zone Variation (Most Likely):**
   - Each motor has slightly different minimum PWM threshold
   - Motor B requires higher PWM to overcome internal friction
   - Normal manufacturing variation in DC motors

2. **L298N Channel Asymmetry:**
   - Channel A and Channel B may have slightly different characteristics
   - Voltage drops could differ between channels

3. **Motor Physical Differences:**
   - Internal resistance variation
   - Gearbox friction differences (even in new motors)
   - Bearing tolerance variations

**Status:** Not a defect - normal motor variation requiring calibration

===== Compensation Strategy Implemented =====

==== Asymmetric PWM Calibration ====

**Principle:** Apply higher PWM to Motor B to compensate for slower response

**Implementation in Code:**

```python
# Modified motor control functions to accept independent speeds
def motor_forward(pwm_a, pwm_b, speed_a, speed_b=None):
    """Allow different speeds for each motor"""
    if speed_b is None:
        speed_b = speed_a  # Default: same speed
    
    GPIO.output(IN1, GPIO.HIGH)  # Motor A forward
    GPIO.output(IN2, GPIO.LOW)
    GPIO.output(IN3, GPIO.LOW)   # Motor B forward (inverted)
    GPIO.output(IN4, GPIO.HIGH)
    pwm_a.ChangeDutyCycle(speed_a)
    pwm_b.ChangeDutyCycle(speed_b)
```

**Calibration Values:**

|| Test Scenario | Motor A PWM | Motor B PWM | Offset |
|---------------|-------------|-------------|--------|
| Test 1 (Speed Calibration) | 20% → 40% | 30% → 50% | +10% |
| Test 2 (Emergency Brake) | 30% → 50% | 30% → 50% | 0% |
| Test 3 (Differential Drive) | 50% | 50% | 0% |

**Rationale:**
* Test 1: Maximum compensation (+10%) to observe synchronization improvement
* Test 2: Equal speeds to test at higher PWM range (where dead zone is not an issue)
* Test 3: Equal speeds sufficient at 50% PWM

===== Lessons Learned =====

==== Technical Insights ====

1. **Motor Dead Zone is Real:**
   * DC motors have minimum PWM threshold to overcome static friction
   * Threshold varies between motors (even same model)
   * Must test and calibrate individually

2. **Asymmetric Calibration Works:**
   * Different PWM values per motor can compensate for physical differences
   * Software compensation more practical than hardware replacement
   * Acceptable solution for manufacturing variation

3. **Instant Brake Preferred for Safety:**
   * Gradual brake looks smooth but slower response
   * Instant brake better for emergency stops
   * Can implement gradual brake for normal deceleration later

4. **Differential Drive Requires Traction:**
   * Turning effectiveness directly proportional to rear wheel grip
   * Low friction = excessive slippage = poor control
   * Front wheel friction must be LOW for differential drive to work

5. **Power Margin Important:**
   * Testing at 50% max PWM leaves room for adjustments
   * Can increase speed later without hardware changes
   * Conservative approach prevents motor/driver overheating

---

==== Project Management Insights ====

1. **Iterative Testing Essential:**
   * Started with assumed parameters (5-30% PWM)
   * Discovered dead zone issue during testing
   * Adjusted to 20-50% based on observations
   * Result: Functional system through iteration

2. **Hardware Variation Expected:**
   * New motors don't guarantee identical behavior
   * Calibration phase necessary for any system
   * Document baseline for future reference

3. **Mechanical Issues Impact Software:**
   * Best software calibration can't overcome lack of traction
   * Hardware improvements (rubber on wheels) necessary
   * Software and hardware must work together
