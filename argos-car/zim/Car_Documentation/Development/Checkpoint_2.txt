==== Critical Challenge Identified: Wheel Traction ====

===== The Traction Problem =====

**Issue:**
* Front wheels are fixed (non-rotating)
* Rear wheels need adequate traction for differential drive
* 3D printed plastic wheels have insufficient grip on floor
* Car slides instead of turning cleanly

**Initial Solution Attempt: Rubber Strips**
```
Method: Add rubber/foam strips to rear wheels
Problem encountered:
├── Rubber strips would not stay attached
├── Adhesive insufficient for rotating wheels
└── Strips would peel off during operation
```

**Second Solution Attempt: Screws to Fix Rubber**
```
Method: Use screws to mechanically attach rubber to wheels
Result: PARTIAL SUCCESS with issues

Improvements:
✅ Traction significantly improved
✅ Rubber stays attached to wheels
✅ Differential drive works better

Problems created:
❌ Screws protrude slightly from wheel surface
❌ At certain points during rotation, screws contact floor
❌ When screw touches floor: wheel slips/skids
❌ Audible noise when screw hits floor surface
❌ Unpredictable behavior during turns
```

===== Differential Drive Calibration Requirements =====

==== Pending Fine-Tuning Tasks ====

**The Challenge:**
> Differential drive requires precise calibration of **three interdependent variables:**

**Variable 1: Time Duration**
* How long motors run in opposite directions
* Directly affects turning angle
* Needs testing for 45°, 90°, 180° turns

**Variable 2: Motor Power/Speed**
* PWM duty cycle percentage for each motor
* Higher speed = sharper turns but less control
* Lower speed = smoother turns but slower response
* Balance needed for demo-quality movements

**Variable 3: Wheel Traction**
* Friction coefficient between tire and floor
* Currently unknown with DIY rubber + screws
* Will change significantly with new rubber tires
* Must be re-tested after tire installation

**Interdependency:**
```
Time × Power × Traction = Turning Angle

Change ONE variable → ALL others must be recalibrated

Example:
├── Current Setup: 2 seconds @ 50% power + screw tires = ~90° turn
└── New Rubber Tires: ??? seconds @ ??? power = 90° turn
    (Must be determined experimentally)
```

===== Pending Discussions Before Stage 1 Final =====

==== Open Questions to Resolve ====

**Topic 1: LCD Display Strategy**

**Question:** Where should LCD 1602 be connected?

**✅ DECISION MADE: LCD on Raspberry Pi**

**Configuration Finalized:**
```
LCD 1602 I2C Module (4-pin interface):
├── VCC  → 4x AA Batteries (+4.8V) - Shared power with sensors
├── GND  → 4x AA Batteries (GND) + Raspberry Pi Pin 6 (common GND)
├── SDA  → Raspberry Pi Pin 3 (GPIO 2 - SDA1)
└── SCL  → Raspberry Pi Pin 5 (GPIO 3 - SCL1)
```

**Why This Decision:**
* ✅ LCD has I2C module (only needs 4 pins, not 16)
* ✅ Uses dedicated I2C pins (3 and 5) - hardware pull-up resistors included
* ✅ Direct Pi control - no communication delay
* ✅ Minimal GPIO usage (only 2 pins for I2C)
* ✅ Can display real-time status during demo

**Important Discovery:**
* LCD purchased has RGB backlight capability
* 16 pins soldered on side are NOT needed (I2C module handles everything)
* Old soldered wires will be removed/isolated before installation
* RGB color control to be tested (green for normal, red for attack)

**Installation Plan:**
* Phase: After tire calibration (March 2026)
* Action: Remove unnecessary 16-pin soldered connections
* Test: Basic text display first, then RGB backlight control
