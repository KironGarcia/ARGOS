====== Raspberry Pi Zero W - Pin Mapping for ARGOS Project ======
Created Monday 10 February 2026

===== Overview =====

This document maps all GPIO pins used in the ARGOS project (Stage 1 Car + Stage 2 Alexa China). It serves as a reference for the ESP32 "Alexa China" to understand which pins are available and which are already in use.

**Raspberry Pi Model:** Zero W  
**Total GPIO Pins:** 40 physical pins  
**GPIO Available:** 26 programmable GPIO pins  
**Purpose:** Car control, sensor monitoring, security algorithms, UART communication with ESP32

===== Complete Pin Usage Map =====

==== Motor Control (L298N) ====

**Motor A Control:**
```
Physical Pin    GPIO    Function              Component
├── Pin 37      GPIO 26  ENA (PWM Enable)      L298N Motor A
├── Pin 11      GPIO 17  IN1 (Direction 1)    L298N Motor A
└── Pin 13      GPIO 27  IN2 (Direction 2)    L298N Motor A
```

**Motor B Control:**
```
Physical Pin    GPIO    Function              Component
├── Pin 38      GPIO 20  ENB (PWM Enable)      L298N Motor B
├── Pin 15      GPIO 22  IN3 (Direction 1)    L298N Motor B
└── Pin 16      GPIO 23  IN4 (Direction 2)    L298N Motor B
```

**Ground Connections:**
```
Physical Pin    Function              Component
├── Pin 6       GND                   L298N (shared)
└── Pin 14      GND                   L298N (shared)
```

**Notes:**
* ENA/ENB use GPIO 26/20 (General Purpose pins, optimized for PWM)
* IN1-IN4 control motor direction (forward/backward)
* Multiple GND pins used for current distribution

**Current Configuration:**
```
Physical Pin    GPIO    Function              Component
├── Pin 18      GPIO 24  TRIG (Trigger)       HC-SR04
├── Pin 19      GPIO 10  ECHO (Echo)           HC-SR04
└── Pin 9       GND      Ground                HC-SR04
```

**Future Configuration (Planned):**
```
Physical Pin    GPIO    Function              Component
├── Pin 35      GPIO 19  TRIG (Trigger)       HC-SR04
├── Pin 36      GPIO 16  ECHO (Echo)           HC-SR04
└── Pin 9       GND      Ground                HC-SR04
```

**Notes:**
* Sensor powered by 5V powerbank (not Pi GPIO)

==== LCD Display (1602 I2C) ====

**I2C Interface:**
```
Physical Pin    GPIO    Function              Component
├── Pin 3       GPIO 2   SDA1 (I2C Data)       LCD 1602
├── Pin 5       GPIO 3   SCL1 (I2C Clock)      LCD 1602
└── Pin 6       GND      Ground                LCD 1602 (shared)
```

**Power:**
```
LCD VCC → 4x AA Batteries (+4.8V)
LCD GND → 4x AA Batteries (GND) + Pi Pin 6 (common GND)
```

**Notes:**
* LCD has I2C module (only needs 2 GPIO pins)
* Uses dedicated I2C pins (hardware pull-up resistors)
* RGB backlight control via I2C commands
* Power from AA batteries (shared with sensors)

==== UART Communication (ESP32 "Alexa China") ====

**UART Interface:**
```
Physical Pin    GPIO    Function              Component
├── Pin 8       GPIO 14  TX (Transmit)         ESP32 RX
├── Pin 10      GPIO 15  RX (Receive)          ESP32 TX
└── Pin 6       GND      Ground                ESP32 GND (shared)
```

**Communication Details:**
* Baud rate: 115200 bps
* Protocol: JSON messages
* Direction: Bidirectional (commands and status)
* Purpose: Voice command transmission from ESP32 to Pi

**Notes:**
* Pin 10 (GPIO 15) was previously used for ENA, moved to Pin 37
* UART pins have hardware support (not software serial)
* Common GND essential for reliable communication

===== Power Distribution =====

==== Power Sources ====

**Source 1: 12V Battery (Motors)**
```
12V Battery
├── +12V → L298N "+12V" pin
└── GND  → L298N "GND" + Pi Pin 6/14 (common GND)
```

**Source 2: 5V Powerbank (Raspberry Pi + Sensor)**
```
5V Powerbank
├── +5V  → Raspberry Pi micro-USB
├── +5V  → HC-SR04 VCC
└── GND  → HC-SR04 GND + Pi Pin 9 (common GND)
```

**Source 3: 4x AA Batteries (Sensors + LCD + ESP32)**
```
4x AA Batteries (4.8V)
├── +4.8V → ESP32 VIN
├── +4.8V → LCD VCC
└── GND   → ESP32 GND + LCD GND + Pi Pin 6 (common GND)
```

**Common Ground:**
* All three power systems share GND reference
* Connected via Raspberry Pi GND pins (6, 9, 14)
* Essential for reliable communication and sensor operation

===== Pin Selection Rationale =====

==== Why These Specific Pins? ====

**Motor Control (GPIO 26/20):**
* ✅ General Purpose pins (no special functions)
* ✅ Good PWM signal quality
* ✅ Avoids conflicts with SPI/I2C/UART

**I2C LCD (GPIO 2/3):**
* ✅ Dedicated I2C pins (hardware pull-up resistors)
* ✅ Standard I2C interface
* ✅ No software configuration needed

**UART ESP32 (GPIO 14/15):**
* ✅ Dedicated UART pins (hardware support)
* ✅ Reliable communication
* ✅ Standard serial interface

**Sensor (GPIO 19/16 future):**
* ✅ General Purpose pins
* ✅ Clean signal paths
* ✅ No special function conflicts

===== Troubleshooting Guide =====

==== Common Pin Issues ====

**Problem: Motor not responding**
* Check: ENA/ENB pins (GPIO 26/20) configured correctly
* Check: IN1-IN4 pins (GPIO 17/27/22/23) logic levels
* Check: GND connections (Pins 6, 14)

**Problem: LCD not displaying**
* Check: I2C pins (GPIO 2/3) connected correctly
* Check: I2C address (0x27 or 0x3F)
* Check: Power supply (4.8V from AA batteries)

**Problem: UART communication fails**
* Check: TX/RX not swapped
* Check: Common GND connection
* Check: Baud rate matches (115200)
* Check: Pins 8 and 10 (GPIO 14/15)

**Problem: Sensor readings erratic**
* Check: TRIG/ECHO pins (GPIO 24/10)
* Check: Voltage divider on ECHO pin
* Check: GND connection (Pin 9)
