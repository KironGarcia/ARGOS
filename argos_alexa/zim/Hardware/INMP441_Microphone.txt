====== INMP441 Microphone Module ======
Created Monday 10 February 2026

===== Overview =====

The INMP441 is a high-performance, low-power digital MEMS microphone with I2S interface. It provides professional-quality audio capture for the ESP32 "Alexa China" voice recognition system.

**Role in Project:** Voice input for wake word detection and command recognition  
**Interface:** I2S (Inter-IC Sound) digital interface  
**Connection:** ESP32 I2S pins  
**Power:** 3.3V from ESP32

===== Pin Configuration =====

==== INMP441 Module Pinout ====

**Typical Module Layout:**
```
INMP441 Module
├── VDD  → Power supply (3.3V)
├── GND  → Ground
├── WS   → Word Select (LRCLK - Left/Right Clock)
├── SCK  → Serial Clock (BCLK - Bit Clock)
└── SD   → Serial Data (DOUT - Data Output)
```

**Pin Functions:**
* **VDD:** Power supply input (1.5V - 3.6V, typically 3.3V)
* **GND:** Ground reference
* **WS (Word Select):** Frame synchronization signal (LRCLK)
  - HIGH = Left channel
  - LOW = Right channel
  - For mono: toggles at sample rate
* **SCK (Serial Clock):** Bit clock signal (BCLK)
  - Determines data transfer rate
  - Frequency = Sample rate × Bits per sample × Channels
* **SD (Serial Data):** Digital audio data output (DOUT)
  - 24-bit audio samples
  - Left-justified format
  - MSB first

===== I2S Interface Details =====

==== I2S Protocol ====

**I2S (Inter-IC Sound) is a serial bus interface standard for digital audio.**

**Signal Description:**
* **WS (Word Select):** Frame synchronization
  - Defines left/right channel
  - Toggles at sample rate frequency
  - For mono: still toggles (use one channel)
* **SCK (Serial Clock):** Bit clock
  - Synchronizes data transfer
  - Frequency = Sample rate × Bits × Channels
  - Example: 16 kHz × 24 bits × 2 channels = 768 kHz
* **SD (Serial Data):** Audio data stream
  - 24-bit samples (left-justified)
  - MSB first
  - Continuous data stream

**Timing Example (16 kHz, 24-bit, mono):**
* Sample rate: 16,000 samples/second
* Bits per sample: 24
* Channels: 1 (mono, but I2S uses 2 slots)
* SCK frequency: 16,000 × 24 × 2 = 768,000 Hz (768 kHz)
* WS frequency: 16,000 Hz (matches sample rate)

===== Troubleshooting Guide =====

==== Common Issues ====

**Problem: No audio data received**
* Check: I2S pin connections (WS, SCK, SD)
* Check: Power supply (3.3V on VDD)
* Check: GND connection
* Check: I2S configuration in ESP32 code
* Check: Sample rate matches INMP441 capability

**Problem: Distorted audio**
* Check: Power supply voltage (should be 3.3V stable)
* Check: Sample rate configuration
* Check: Bit depth settings
* Check: I2S timing (clock frequency)

**Problem: Low sensitivity**
* Check: Microphone orientation
* Check: Distance from sound source
* Check: Background noise levels
* Check: I2S gain settings (if available)

**Problem: High noise level**
* Check: Power supply quality (clean 3.3V)
* Check: GND connection (common ground with ESP32)
* Check: Cable length (keep I2S wires short)
* Check: Electromagnetic interference (away from motors)
