==== Initial Testing Problems ====

==== Problem 1: LEDs Not Lighting ====

**Symptom:**
- Code uploaded successfully
- No LED activity observed
- Only ESP32 status LED visible

**Investigation:**
- Verified power supply: 3.3V measured with multimeter ✅
- Verified breadboard rails: Positive and negative confirmed ✅
- Verified LED polarity: Long leg (anode) and short leg (cathode) correct ✅
- Verified resistors: Properly connected ✅
- Verified GPIO connections: Jumpers connected to correct pins ✅

**Root Cause Identified:**
- **Poor contact in breadboard** - LEDs not making proper electrical contact
- Loose connections preventing current flow

**Solution:**
- Reseated LEDs in breadboard with better contact
- Ensured firm connection of all components
- Result: ✅ LEDs began working immediately

**Lesson Learned:**
- Breadboard contact quality is critical
- Always verify physical connections before troubleshooting code
- Multimeter measurements don't guarantee good breadboard contact

- Some PWM functions not available in this version

**Solution:**

- More compatible across ESP32 Arduino core versions
- Result: ✅ Code compiled and worked perfectly

**Final Working Code:**

- Compatible with current ESP32 Arduino core

===== Lessons Learned =====

**1. Breadboard Quality Matters:**
- Poor contact can prevent components from working
- Always verify physical connections before code troubleshooting
- Reseating components often solves connection issues

**2. Pin Selection is Critical:**
- Not all GPIO pins are equal on ESP32
- Strapping pins (GPIO 0, 2, 4, 12, 15) can cause boot issues
- General-purpose pins (GPIO 5, 18, 19, etc.) are safer

**3. Code Compatibility:**
- Different ESP32 Arduino core versions have different APIs
- `analogWrite()` is more compatible than `ledcSetup()`
- Always test with simple code first

**4. Systematic Troubleshooting:**
- Verify power supply first
- Check physical connections
- Then troubleshoot code
- Multimeter is essential tool
