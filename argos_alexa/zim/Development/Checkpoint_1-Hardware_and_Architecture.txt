==== Test sketches and results ====

* ''alexa-architecture/src/1test/1test.ino'' — Combined menu (mic + speaker + LEDs). First integration test. Revealed LED interference issue.
* ''alexa-architecture/src/teste_só_microfone/teste_só_microfone.ino'' — Isolated mic test, 10 s capture + analysis. **Result: MICROFONE FUNCIONOU** (nível médio: 7361, pico: 32768, amostras: 160768).
* ''alexa-architecture/src/teste_só_luces/teste_só_luces.ino'' — Isolated LED test, both LEDs fade together 5 s cycle. Passed.
* Speaker tested via 1test option 2: sweep tone 200 Hz → 2000 Hz. **Passed. Volume strong.**

==== Decision: LEDs removed from final hardware ====

LEDs were tested and worked correctly (GPIO 5 yellow, GPIO 18 red). However, they were **removed from the project** for the following reasons:

* Did not provide the visual impact needed for the demo.
* Added complexity and wiring without clear benefit.
* Caused audible interference (hiss) in the speaker when driven simultaneously.

**Final hardware: ESP32 + INMP441 (mic) + PAM8403 + 4Ω 3W speaker. No LEDs.**

===== 2. Architecture Defined: Voice Communication System =====

==== Concept ====

The Alexa China device (ESP32) acts as the **audio peripheral**: it captures voice and plays back responses. The Raspberry Pi Zero W (argos-car brain) acts as the **intelligent coordinator**: it receives detected intents via WiFi, decides the action (voice response and/or car command), and returns audio to the ESP32.

No internet required. Everything runs on an isolated local WiFi network.

==== Communication flow ====

```
User speaks keyword
  → INMP441 captures audio
  → ESP32 runs Edge Impulse model locally (~100–200 ms inference)
  → Keyword detected: ESP32 sends HTTP POST to RPi {"intent": "keyword_name"}
  → RPi Flask server receives intent
  → RPi selects pre-recorded .wav response (or triggers car command)
  → RPi returns .wav audio in HTTP response body
  → ESP32 receives .wav, plays via DAC (GPIO 25) → PAM8403 → speaker
```

Total estimated latency: **0.5–1.5 seconds** (local WiFi, no internet).

==== Technology stack ====

| Layer | Technology | Notes |
|---|---|---|
| Keyword detection | **Edge Impulse** (TFLite model, Arduino library export) | Train in browser; 8–15 keywords max on ESP32 Dev; ~200 ms inference |
| ESP32 WiFi | ESP32 native WiFiClient + HTTPClient | Connect to isolated local network |
| ESP32 → RPi | **HTTP POST** JSON `{"intent": "name"}` | <50 ms on local WiFi |
| RPi server | **Python Flask** on port 5000 | Lightweight, easy to maintain |
| RPi TTS/audio | **Pre-recorded .wav files** (recommended for demo stability) | Generated with espeak-ng or online TTS; stored on RPi disk |
| RPi → ESP32 | .wav file in HTTP response body | Same HTTP connection, no second round-trip |
| ESP32 playback | **ESP8266Audio library** + AudioOutputDAC (GPIO 25) | Supports WAV streaming from HTTP or SD/SPIFFS |

==== Car control via voice ====

The same intent system controls the car. The RPi Flask server handles each intent independently:

* Conversational intents (e.g. "saludo", "origen_argos") → RPi returns .wav audio only.
* Car control intents (e.g. "iniciar_presentacion", "parar") → RPi launches/stops the car program (subprocess) AND returns a .wav confirmation audio.
* Example: user says "iniciar presentação" → ESP32 detects → RPi runs argos-car/src/menu_demo.py + plays "Iniciando presentação do ARGOS Car...".

==== Keyword list (10 intents — defined for the demo) ====

| # | Spoken phrase | Intent name | Action |
|---|---|---|---|
| 1 | "ARGOS iniciar presentación" | ''iniciar_presentacion'' | RPi launches argos-car demo program (menu_demo.py) + plays confirmation audio |
| 2 | "ARGOS quién eres" | ''quien_eres'' | Plays pre-recorded ARGOS identity presentation audio |
| 3 | "ARGOS derecha" | ''mover_derecha'' | RPi sends right-turn command to car + plays "girando a la derecha" |
| 4 | "ARGOS izquierda" | ''mover_izquierda'' | RPi sends left-turn command to car + plays "girando a la izquierda" |
| 5 | "ARGOS adelante" | ''mover_adelante'' | RPi sends forward command to car + plays "avanzando" |
| 6 | "ARGOS atrás" | ''mover_atras'' | RPi sends reverse command to car + plays "retrocediendo" |
| 7 | "ARGOS para" | ''parar'' | RPi stops car motors + plays "deteniendo el sistema" |
| 8 | "ARGOS cuenta un chiste" | ''contar_chiste'' | Plays long pre-recorded joke audio (from SD card) |
| 9 | "ARGOS toca una música" | ''tocar_musica'' | Plays .mp3 or .wav music file from SD card |
| 10 | "ARGOS fin" | ''fin_conversacion'' | Closes demo mode + plays "hasta pronto" |

**Total: 10 keywords.** Within safe range for ESP32 Dev Module (520 KB SRAM). Edge Impulse model with 10 classes + background noise is feasible with ~60–80 samples per keyword.

==== SD card module — confirmed addition to the project ====

The SD card module is **confirmed as a required component** for the next hardware revision. Reasons:

* Long audio responses (ARGOS identity, jokes, music) do not fit in SPIFFS (~1.5 MB free).
* Music playback (.mp3/.wav) requires significant storage.
* SD card (2–8 GB) gives effectively unlimited space for all .wav responses + future models.
* ESP8266Audio supports ''AudioFileSourceSD'' natively.

**SPI pinout for SD module on ESP32 (pins freed after LED removal):**

| SD Module Pin | ESP32 Pin | Notes |
|---|---|---|
| MOSI | D23 (GPIO 23) | Standard SPI MOSI |
| MISO | D19 (GPIO 19) | Standard SPI MISO |
| SCK | D18 (GPIO 18) | Standard SPI SCK — free (was red LED, now removed) |
| CS | D5 (GPIO 5) | Chip select — free (was yellow LED, now removed) |
| VCC | 3V3 | SD module power |
| GND | GND | Common ground |

⚠️ **Verify before wiring:** GPIO 18 (D18) and GPIO 5 (D5) are free because the LEDs were removed. Confirm no other component uses these pins before connecting the SD module.

**Files to store on SD card:**
* ''argos_identity.wav'' — ARGOS self-presentation (long, ~15–30 s)
* ''iniciar_presentacion.wav'' — "Iniciando presentação do ARGOS Car..."
* ''derecha.wav'', ''izquierda.wav'', ''adelante.wav'', ''atras.wav'', ''para.wav'' — Short movement confirmations
* ''chiste.wav'' — Pre-recorded joke (long, ~20–30 s)
* ''musica.wav'' or ''musica.mp3'' — Demo music track
* ''fin.wav'' — "Hasta pronto, soy ARGOS."
