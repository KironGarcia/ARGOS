===== Component Overview =====

==== Hardware Components ====

**ESP32 Development Board:**
* Main controller for voice system
* I2S interface for microphone
* DAC output for speaker
* UART communication with Raspberry Pi
* Power: 4x AA batteries (4.8V)

**INMP441 Microphone (I2S):**
* High-quality digital microphone
* I2S interface (3 wires: VDD, GND, WS, SCK, SD)
* Low power consumption
* Professional audio quality

**PAM8403 Amplifier Module:**
* Class-D audio amplifier
* 3W output power
* Low voltage operation (2.5V-5.5V)
* Perfect for ESP32 DAC output

**Speaker 4Ω 3W:**
* Matched to amplifier output
* Good frequency response
* Compact size for car integration

**Raspberry Pi Zero W:**
* Car control and security monitoring
* Receives voice commands via UART
* Executes motor control
* Runs ARGOS security algorithms

---

===== System Architecture =====

==== Communication Flow ====

**Normal Operation:**
```
User Voice → INMP441 Mic → ESP32 (Wake Word Detection)
         → ESP32 (Command Recognition)
         → ESP32 → UART → Raspberry Pi
         → Pi (Motor Control / Status)
         → ESP32 ← UART ← Pi (Response)
         → ESP32 → PAM8403 → Speaker
         → User hears: "Moving forward"
```

**Attack Sequence:**
```
Attacker → Exploits ESP32 (telnet/credentials)
        → Gains ESP32 shell access
        → Injects commands via UART
        → Raspberry Pi receives malicious commands
        → Car moves without user permission
        → ARGOS detects anomaly
        → Emergency brake activated
        → ESP32 isolated
        → System recovered
```

===== Key Design Decisions =====

==== Why ESP32 for Voice? ====

**Advantages:**
* ✅ Dedicated I2S interface for microphone
* ✅ Built-in DAC for audio output
* ✅ Low power consumption
* ✅ Perfect for "cheap IoT device" narrative
* ✅ Offloads voice processing from Raspberry Pi

**Alternative Considered:**
* Bluetooth headset → Rejected (less dramatic, harder to show vulnerability)

---

==== Why I2S Microphone? ====

**INMP441 Benefits:**
* ✅ Digital interface (no ADC noise)
* ✅ High quality audio capture
* ✅ Low power consumption
* ✅ Professional-grade component
* ✅ Easy ESP32 integration

**Alternative Considered:**
* Analog microphone → Rejected (lower quality, more noise)

---

==== Why PAM8403 Amplifier? ====

**PAM8403 Benefits:**
* ✅ Class-D efficiency (low power)
* ✅ Matches ESP32 voltage range
* ✅ 3W output sufficient for demo
* ✅ Small form factor
* ✅ Cost-effective

**Alternative Considered:**
* Direct speaker connection → Rejected (ESP32 DAC too weak)
